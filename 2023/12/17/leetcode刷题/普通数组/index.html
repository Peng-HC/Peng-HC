  • -104 <= nums[i] <= 104

示例 1：

示例 2：

提示：

  • 1 <= intervals.length <= 104
  • intervals[i].length == 2
  • 0 <= starti <= endi <= 104

解题方法

思路：

将各个区间按照start排序，比较前一个区间和当前区间是否有重叠(intervals[i][0]<=intervals[i-1][1])，若有，则修改当前区间的start和end值，使其包含这两个区间；若没有，则将前一个区间添加到res中；最后，需要将intervals[n-1]，即最后一个区间添加到res中（因为每次添加都是前一个区间，会漏掉最后一个区间）。

示例 2:

提示：

  • 1 <= nums.length <= 105
  • -231 <= nums[i] <= 231 - 1
  • 0 <= k <= 105

进阶：

  • 尽可能想出更多的解决方案，至少有 三种 不同的方法可以解决这个问题。
  • 你可以使用空间复杂度为 O(1) 的 原地 算法解决这个问题吗？

解题方法

思路：三次翻转。用于解决数组元素向右循环移位的问题

例如：nums = [1,2,3,4,5,6,7], k = 3

  1. 第一次整体翻转：[7,6,5,4,3,2,1]
  2. 第二次前k个元素翻转：[5,6,7,4,3,2,1]
  3. 第三次后n-k个元素翻转：[5,6,7,1,2,3,4]
